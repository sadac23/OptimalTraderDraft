はい、AWSの無料プランを利用してC#コンソールアプリを定期実行するシステムを構築することは可能です。以下の手順で実現できます。

1. AWS Lambdaを使用する方法
AWS Lambdaはサーバーレスコンピューティングサービスで、コードを実行するためのインフラストラクチャを管理する必要がありません。
無料プランには一定の無料使用枠が含まれています。

手順:
Lambda関数の作成:

AWS Management Consoleにログインし、Lambdaサービスに移動します。
「関数の作成」をクリックし、ランタイムとして「.NET Core」を選択します。
必要なコードをアップロードします。

CloudWatch Eventsの設定:

AWS Management ConsoleでCloudWatchに移動します。
「ルールの作成」をクリックし、スケジュールを選択します。
Cron式またはレート式を使用して、関数の実行頻度を設定します。
ターゲットとして先ほど作成したLambda関数を選択します。

2. AWS EC2を使用する方法
AWS EC2インスタンスを使用して、定期的にC#コンソールアプリを実行することも可能です。無料プランには一定の無料使用枠が含まれています。

手順:
EC2インスタンスの作成:

AWS Management Consoleにログインし、EC2サービスに移動します。
「インスタンスの起動」をクリックし、無料利用枠に含まれるインスタンスタイプ（t2.microなど）を選択します。
必要な設定を行い、インスタンスを起動します。

C#コンソールアプリのデプロイ:

インスタンスにSSHまたはRDPで接続します。
必要なランタイム（.NET Coreなど）をインストールします。
C#コンソールアプリをインスタンスにアップロードします。
定期実行の設定:

Windowsの場合、タスクスケジューラを使用して定期実行を設定します。
Linuxの場合、cronジョブを設定して定期実行を設定します。

まとめ
AWSの無料プランを利用してC#コンソールアプリを定期実行するシステムを構築するには、AWS LambdaまたはAWS EC2を使用する方法があります。
Lambdaはサーバーレスで管理が簡単ですが、EC2はより柔軟性があります。どちらの方法も無料プラン内で利用可能です。
